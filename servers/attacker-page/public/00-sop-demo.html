📐 What defines an origin?

An origin is the combination of scheme + host + port. All three must match for two URLs to be considered same-origin. If any one differs, they are cross-origin and SOP applies.

🔑 CORS unlocks cross-origin reads (selectively)

CORS is the browser's mechanism for a server to say: "I permit this specific origin to read my responses." It is opt-in. Without it, SOP blocks all cross-origin reads. With it, only explicitly allowed origins get access.

✈️ Preflights — when the browser asks permission first

For "non-simple" requests (custom headers, PUT/DELETE, etc.), the browser sends an OPTIONS request first — the preflight — to ask if the real request is allowed. Only if the server approves does the browser send the actual request.

Simple requests (GET/POST with standard headers) skip the preflight. That's why many CORS bugs are exploitable with a plain GET.
