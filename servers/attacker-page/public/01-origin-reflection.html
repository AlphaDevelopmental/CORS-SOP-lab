Bugs #1 (wildcard) and #2 (origin reflection) — live in your browser
Attacker scenario: You control this page at . The victim bank API runs at http://localhost:3000. A victim user has a session cookie set by the bank. They visit your malicious page. Your JavaScript runs silently and exfiltrates their account data — without them seeing anything.

BUG #1 Wildcard Origin (Access-Control-Allow-Origin: *)

The server sends * as the allowed origin. Any website on the internet can read this response. While browsers block * combined with credentials, this route still leaks all its data to any unauthenticated cross-origin caller.
Request Origin: http://localhost:3002 Response header: Access-Control-Allow-Origin: * ← any origin allowed

BUG #2 Origin Reflection (your snippet from class)

The server reads req.headers.origin and echoes it directly into Access-Control-Allow-Origin. Combined with Allow-Credentials: true, any website becomes a trusted origin. The browser sends the victim's session cookie, and the attacker's page receives credentialed account data.
Request Origin: http://localhost:3002 ← we are the attacker Response header: Access-Control-Allow-Origin: http://localhost:3002 ← reflected! Response header: Access-Control-Allow-Credentials: true ← cookies included!
